Upload Benchmark

Every benchmark that should be run on the Hobbit platform needs to be uploaded to it. This uploading comprises several steps. Note that a user profile is necessary to have the right to upload something to the platform.

  1. Before uploading, the benchmark has to be implemented in a way that it fits into the platform and is able to interact with the paltform controller. The general development of a benchmark is described here.
  2. The second step is the creation of at least one project in the gitlab instance of the platform. For that you can use the same account as for the platform itself. (The project should have the same name as the system. Please have a look at the image upload and its hints regarding the project name.)
  3. After creating the git project(s), you need to push the Docker image(s) to the git project(s) as described here.
  4. The final step is to put the meta data describing the benchmark into a file called benchmark.ttl and push it to one of the git projects. The format of the file is described here.

All steps are described in detail in our wiki.
